


특정 사이트의 고유 주소를 도메인이라 함

특정 사이트 접속하여 홈페이지를 띄우기 위해서는
소스코드가 들어있는 서버로 접속해야 함.

고유주소가 있는데, 일련의 숫자 조합으로 구성
ex) 210.89.160..~

사람이 기억하기 쉽게
특정 문자로 매핑하여 => DOMAIN NAME SYSYEM
주소창에 도메인을 입력하면 결국 
고유 주소로 가도록 함.

-1. 도메인 구매시

-구매하려는 도메인이 구매할 수 있는지..
-동일한 도메인을 여러 사람이 가질 수 없음
-CAFE24 도메인 검색 -> 원하는 도메인을 치면
등록 여부가 나옴

-되도록이면 .COM을 누군가 구매하고 .KR, .CO.KR은 구매가 가능하다면
-사지 마라

-전세계적으로 사용하기 위해 .COM으로 구매할 수 있는 도메인이 좋음

-또한 .COM을 샀다면 KR.. CO.KR도 사는게 경쟁업체를 막을 수 있음

.com , .co , .co.kr 이런 것들은 registry라 부르고 사기업이 운영함

-홍보는 열심히 하는데 고객은 다른 곳으로...

정리 : registry는 ip 주소와 도메인을 연결함 => do

-2. 도메인 등록 시

주요정보를 입력하게 됨
소유자, 생년월일, 주소, EAMIL, 휴대폰 등..

본인이 구매하는 것이 중요하다. 업체를 통해 대리 구매를 하게 될 경우가 있는데,
도메인 구매 명의를 고객의 명의로 하면 상관없으나, 업체의 명의로 해버리면
나중에 홈페이지를 옮긴다면 이전할 때 소유권에 대해 문제가 생길 수 있음.


호스팅이란?

- 홈페이지 제작 시 소스 코드를 제작하게 됨
- 홈페이지를 제작한다는 것은, 이를 동작할 수 있는 소스 코드를 만드는 것임
- 저장하려면 공간이 필요한데, 자신의 컴퓨터에 운영하려면 365일 계속 작동을 해야하고
- 보안문제, 전력 문제 등이 있으므로
- 속도도 빠르고 보안도 좋고 365일 운영가능한 서버에 올리고자 하는 홈페이지를 올리는 것이 좋다

- 이런 서버를 제공하는 업체가 호스팅 업체

-WWW(인터넷)을 통해 접속 => 서버에 요청 => 서버가 소스코드를 읽고 => 사용자에게 반환


- 상품이해

- 홈페이지에 파일을 업로드 하거나 용량... 사이트에서 필요로 하는 용량 만큼을 구매
- 모두 파악하기 힘들다면 개발자에게 요청해서 구매를 부탁하면 됨

- 600MB트래픽 용량 -> 얼마나 많은 데이터가 왔다 갔다 하는지, 하루 동안에 600MB만의 데이터 만이 왔다 갔다 할 수 잇다
- 600MB 이상의 트래픽이면 서버가 다운됨

- 가격은 천차만별
- 가격과 서비스는 비례한다.
